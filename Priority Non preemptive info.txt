üîÑ 6. Sort processes by priority
        // Sort processes by priority (lower number = higher priority)
        for (int i = 0; i < n - 1; i++) {
            for (int j = i + 1; j < n; j++) {
                if (pr[i] > pr[j]) {
                    // swap priority
                    int temp = pr[i];
                    pr[i] = pr[j];
                    pr[j] = temp;

                    // swap burst time
                    temp = bt[i];
                    bt[i] = bt[j];
                    bt[j] = temp;

                    // swap process id
                    temp = pid[i];
                    pid[i] = pid[j];
                    pid[j] = temp;
                }
            }
        }

Explanation:

The nested loops sort the processes in ascending order of priority.

If pr[i] > pr[j], it means process i has lower priority (since higher number means lower priority), so we swap them.

Whenever we swap priorities, we also swap:

Corresponding burst times (bt),

And process IDs (pid).

üß† So after sorting, all arrays are aligned properly ‚Äî highest priority process comes first.

After sorting:

P2	3	1
P1	4	2
P3	5	3
‚è±Ô∏è 7. Calculate Waiting Time (WT) & Turnaround Time (TAT)
        // Calculate waiting time and turnaround time
        wt[0] = 0;
        tat[0] = bt[0];
        for (int i = 1; i < n; i++) {
            wt[i] = wt[i - 1] + bt[i - 1];
            tat[i] = wt[i] + bt[i];
        }

Step-by-step:

First process always has 0 waiting time because it starts immediately.

Turnaround time for first process = burst time (since no wait).

For each subsequent process:

Waiting time = sum of burst times of all previous processes.

Turnaround time = its own burst time + its waiting time.

üßÆ Example:

PID	BT	WT	TAT
P2	3	0	3
P1	4	3	7
P3	5	7	12

‚öôÔ∏è Summary of Key Concepts
Concept	Description
Priority Scheduling	Each process has a priority number; lower number means higher priority.
Non-preemptive	Once a process starts, it runs until completion. No interruption.
Waiting Time (WT)	Time process waits in ready queue before execution.
Turnaround Time (TAT)	Total time from arrival to completion (WT + BT).
Sorting	Processes are sorted by priority before calculating WT and TAT.
