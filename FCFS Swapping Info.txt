Algorithm: FCFS (non-preemptive)

Goal: Schedule processes in order of their arrival times and compute:

Completion Time (CT)

Turnaround Time (TAT)

Waiting Time (WT)

Swapping: Used to sort processes by arrival time before execution

Step 3 ‚Äì Sorting by Arrival Time
for (int i = 0; i < n - 1; i++) {
    for (int j = i + 1; j < n; j++) {
        if (at[i] > at[j]) {
            // Swap arrival time
            int temp = at[i];
            at[i] = at[j];
            at[j] = temp;

            // Swap burst time
            temp = bt[i];
            bt[i] = bt[j];
            bt[j] = temp;

            // Swap process ID
            temp = pid[i];
            pid[i] = pid[j];
            pid[j] = temp;
        }
    }
}


üîÅ This section sorts the processes in ascending order of Arrival Time (AT).

FCFS executes processes as they arrive, so sorting ensures correct order.

Swapping is used to maintain alignment between arrival times, burst times, and process IDs.

Example:
Before sort ‚Üí AT = [5, 1, 2]
After sort ‚Üí AT = [1, 2, 5]

Step 4 ‚Äì Calculate Completion Time (CT)
ct[0] = at[0] + bt[0]; // First process
for (int i = 1; i < n; i++) {
    if (at[i] > ct[i - 1]) {
        // CPU is idle until process arrives
        ct[i] = at[i] + bt[i];
    } else {
        ct[i] = ct[i - 1] + bt[i];
    }
}


üßÆ Logic:

For first process: CT = AT + BT

For others:

If CPU becomes idle (next process arrives after previous finishes):

Start new process when it arrives: CT = AT + BT

Otherwise:

Start immediately after last completion: CT = previous CT + BT

Example:

Process	AT	BT	CT
P1	0	3	3
P2	2	4	7
P3	5	2	9
Step 5 ‚Äì Calculate TAT & WT
for (int i = 0; i < n; i++) {
    tat[i] = ct[i] - at[i];
    wt[i] = tat[i] - bt[i];
}


Formulas:

Turnaround Time (TAT) = CT - AT

Waiting Time (WT) = TAT - BT

These are standard scheduling metrics:

Term	Meaning
TAT	Total time a process spends in the system
WT	Time a process waits in the ready queue

üß† Key Takeaways

FCFS is non-preemptive (once process starts, it runs till completion).

Processes are executed in order of arrival.

If CPU becomes idle (gap between CT and next AT), it waits.

This version sorts processes by Arrival Time using swapping, hence the name FCFS_Swapping
